INTRODUCTION                                                    *agentlog-intro*

agentlog.nvim turns AI-agent terminal scrollback into a structured Neovim
buffer. The supported workflows are Codex, Claude Code, and Cursor Agent
scrollback opened from Zellij.

The adapters recognize prompts, responses, actions, command output, file
references, and compact or unified diffs. Rendering is non-destructive:
highlights, extmarks, and virtual text change only the presentation. The
original buffer text remains available for selection, yanking, and inspection.

Recognized diff code receives contextual Tree-sitter highlighting when the
language can be inferred from a file path and its parser is installed.

==============================================================================
REQUIREMENTS                                              *agentlog-requirements*

- Neovim 0.10 or newer.
- A Tree-sitter parser and highlight query for every language that should be
  highlighted inside diffs.

Tree-sitter is optional. Actions, output, file references, and structural diff
highlighting continue to work when a parser is unavailable.

==============================================================================
SETUP                                                            *agentlog-setup*

Calling setup() is optional. Defaults are loaded automatically when the plugin
is present on 'runtimepath'.
>
    require("agentlog").setup({
      -- auto_attach = true, -- Uncomment to enable automatic attachment.
      min_confidence = 0.75,
      adapters = {
        claude = { enabled = true },
        codex = { enabled = true },
        cursor = { enabled = true },
      },
      render = {
        diff_background = true,
        diff_code_padding = 1,
      },
      syntax = {
        enabled = true,
        treesitter = true,
        max_region_lines = 500,
      },
      navigation = {
        wrap = true,
      },
      mappings = {
        enabled = true,
        next_action = "]a",
        previous_action = "[a",
        next_diff = "]d",
        previous_diff = "[d",
        next_response = "]r",
        previous_response = "[r",
        next_file = "]f",
        previous_file = "[f",
        next_error = "]e",
        previous_error = "[e",
        next_hunk = "]h",
        previous_hunk = "[h",
        open_file = "gf",
      },
    })
<
Automatic attachment is disabled by default. When enabled, only `*.dump` files
with a strong Codex, Claude Code, or Cursor Agent signature and enough
independent evidence are attached. Manual attachment remains available for
every buffer.

==============================================================================
USAGE                                                            *agentlog-usage*

Open a Codex, Claude Code, or Cursor Agent scrollback dump and run:
>
    :AgentlogAttach
<
The buffer 'filetype' becomes "agentlog" and the current contents are parsed
and rendered. The text and the buffer's modified state are not changed.

Run |:AgentlogRefresh| after the contents change or after installing a missing
Tree-sitter parser. Use |:AgentlogDetach| to clear all decorations and restore
the filetype that was active before attachment.

When using Zellij, open scrollback in the configured editor and attach after
Neovim starts. Automatic attachment can be enabled with `auto_attach = true`.
Set `vim.b.agentlog_disable = true` before |BufReadPost| to prevent automatic
attachment for one buffer. Manual attachment remains possible.

==============================================================================
COMMANDS                                                      *agentlog-commands*

                                                        *:AgentlogAttach*
:AgentlogAttach
        Detect and attach the supported adapter for the current buffer, build
        its region document, set 'filetype' to "agentlog", and render
        decorations. Calling it again refreshes an attached buffer.

                                                       *:AgentlogRefresh*
:AgentlogRefresh
        Reparse an attached buffer and rebuild every decoration. Rendering is
        idempotent; old extmarks are cleared before new ones are applied.

                                                        *:AgentlogDetach*
:AgentlogDetach
        Clear plugin decorations and restore the previous filetype. Buffer
        contents remain unchanged.

                                                          *:AgentlogNext*
:AgentlogNext {kind}
        Jump to the next region of {kind}. Supported kinds are `action`, `diff`,
        `error`, `file`, `hunk`, and `response`. A command count is accepted; for
        example:
>
            :3AgentlogNext action
<
                                                      *:AgentlogPrevious*
:AgentlogPrevious {kind}
        Jump to the previous region of {kind}. Supported kinds and command
        counts match |:AgentlogNext|.

Use |:checkhealth| agentlog to inspect the Neovim and Tree-sitter runtime.

==============================================================================
NAVIGATION                                                *agentlog-navigation*

Navigation mappings are normal-mode, buffer-local, and installed on attach:

    ]a      Next action.
    [a      Previous action.
    ]d      Next changed diff block.
    [d      Previous changed diff block.
    ]r      Next assistant response.
    [r      Previous assistant response.
    ]f      Next recognized file occurrence.
    [f      Previous recognized file occurrence.
    ]e      Next error or warning.
    [e      Previous error or warning.
    ]h      Next explicit unified-diff hunk.
    [h      Previous explicit unified-diff hunk.
    gf      Open the recognized file under the cursor.

Counts are supported, such as `3]a`. Navigation wraps at the first and last
target by default. Jumps enter the window jump list, so normal jump-list
commands can return to the previous location.

All rows belonging to one structured diff are one navigation target. Source
previews containing only context, such as a Claude `Read`, are not considered
changed diffs. `gf` uses the exact path stored by the adapter. When no adapter
path is present at the cursor, Neovim's native |gf| behavior is used.
File navigation includes context-only previews and groups all regions sharing
one structured file operation into a single target.
Adjacent error and warning rows form one diagnostic target; separate diagnostic
blocks remain independently navigable.
Hunk navigation targets each `@@ ... @@` header within a unified diff. Compact
previews without explicit hunk headers do not produce synthetic targets.

Existing buffer-local mappings take precedence. Detach removes only mappings
installed by agentlog. Disable or customize the defaults through
|agentlog-mappings|.

==============================================================================
RENDERING                                                    *agentlog-rendering*

Turns and actions ~

Codex action headings such as `Ran`, `Edited`, `Added`, `Deleted`, `Explored`,
`Read`, and `Searched` are emphasized. Claude user prompts, responses, tool
calls, output, warnings, progress, and interface metadata receive separate
semantic groups. Cursor Agent actions, todo state, shell output, and interface
rows use the same shared semantic groups. Recognized file paths retain language
and diff metadata.

Diffs ~

Unified diffs, Codex's compact numbered format, Claude `Update` previews, and
Cursor `Edited` previews are supported. Claude `Write` previews are treated as
newly added numbered source without displaying a nonexistent marker. Blank rows
and wrapped result paths between a Claude tool call and its source preview are
supported. Compact diff rows are split into independent visual layers:

- line number;
- `+`, `-`, or context marker;
- optional muted add/delete background;
- virtual padding before the source code;
- Tree-sitter captures for the source language.

`diff_code_padding` is measured in terminal cells, not pixels. Padding uses
inline virtual text and therefore does not affect copied text. Cursor previews
keep their native separator spacing and ignore this option.

Syntax highlighting ~

For each diff, agentlog builds normalized old and new source snapshots. Diff
markers and line numbers are removed only from these internal representations.
Tree-sitter captures are then mapped back to the original buffer columns.

PHP snippets without an opening tag receive a synthetic `<?php` prefix in the
internal snapshot. The prefix is never inserted into the visible buffer.

Languages currently inferred from common extensions include Bash, C, C++,
CSS, Go, HTML, JavaScript, JSON, Lua, Markdown, PHP, Python, Rust, TypeScript,
TSX, Vim, and YAML.

Highlight groups ~

The plugin defines semantic groups that colorschemes and users may override:
>
    AgentlogResponse
    AgentlogAction
    AgentlogActionSuccess
    AgentlogActionFailure
    AgentlogCommand
    AgentlogOutput
    AgentlogMetadata
    AgentlogFile
    AgentlogDiffAdd
    AgentlogDiffDelete
    AgentlogDiffContext
    AgentlogDiffHeader
    AgentlogDiffLineNumber
    AgentlogDiffAddBackground
    AgentlogDiffDeleteBackground
    AgentlogError
    AgentlogWarning
    AgentlogCollapsed
<
Default groups link to standard Neovim highlights. Muted diff backgrounds are
derived from the active colorscheme and reapplied after |ColorScheme| events.

==============================================================================
CONFIGURATION                                                  *agentlog-config*

                                                   *agentlog-auto-attach*
`auto_attach` ~
        Boolean, default false. Detect and attach supported `*.dump` scrollback
        on |BufReadPost| when a supported agent signature is present and the
        confidence threshold is met.

                                                *agentlog-min-confidence*
`min_confidence` ~
        Number from 0 to 1, default 0.75. Minimum detector score required for
        automatic attachment.

                                             *agentlog-adapters-claude*
`adapters.claude.enabled` ~
        Boolean, default true. Enable the built-in Claude Code adapter.

                                              *agentlog-adapters-codex*
`adapters.codex.enabled` ~
        Boolean, default true. Enable the built-in Codex adapter.

                                             *agentlog-adapters-cursor*
`adapters.cursor.enabled` ~
        Boolean, default true. Enable the built-in Cursor Agent adapter.

                                             *agentlog-diff-background*
`render.diff_background` ~
        Boolean, default true. Add muted backgrounds to inserted and deleted
        source text. Line numbers and markers remain independently
        highlighted.

                                           *agentlog-diff-code-padding*
`render.diff_code_padding` ~
        Non-negative integer, default 1. Number of virtual terminal cells
        inserted between the diff marker and source code. Use 0 to disable.
        Cursor previews always keep their native spacing and ignore this value.

                                                *agentlog-syntax-enabled*
`syntax.enabled` ~
        Boolean, default true. Enable contextual syntax highlighting.

                                             *agentlog-syntax-treesitter*
`syntax.treesitter` ~
        Boolean, default true. Use Tree-sitter for normalized diff snapshots.

                                       *agentlog-syntax-max-region-lines*
`syntax.max_region_lines` ~
        Positive integer, default 500. Skip advanced parsing for larger diff
        regions while keeping structural highlighting active.

                                                *agentlog-navigation-wrap*
`navigation.wrap` ~
        Boolean, default true. Continue navigation from the opposite end when
        no later or earlier target exists.

                                                    *agentlog-mappings*
`mappings.enabled` ~
        Boolean, default true. Install agentlog's normal-mode buffer-local
        mappings on attach.

`mappings.next_action`, `mappings.previous_action` ~
        Strings, defaults `]a` and `[a`. Navigate between action regions.

`mappings.next_diff`, `mappings.previous_diff` ~
        Strings, defaults `]d` and `[d`. Navigate between changed diff blocks.

`mappings.next_response`, `mappings.previous_response` ~
        Strings, defaults `]r` and `[r`. Navigate between assistant responses.

`mappings.next_file`, `mappings.previous_file` ~
        Strings, defaults `]f` and `[f`. Navigate between recognized file
        occurrences.

`mappings.next_error`, `mappings.previous_error` ~
        Strings, defaults `]e` and `[e`. Navigate between errors and warnings.

`mappings.next_hunk`, `mappings.previous_hunk` ~
        Strings, defaults `]h` and `[h`. Navigate between explicit unified-diff
        hunk headers.

`mappings.open_file` ~
        String, default `gf`. Open the recognized file at the cursor. Use an
        empty string to disable any individual mapping.

==============================================================================
LUA API                                                            *agentlog-api*

The command behavior is also available through Lua:
>
    local agentlog = require("agentlog")

    agentlog.setup(options)
    agentlog.attach(bufnr)
    agentlog.attach(bufnr, { source = "claude" })
    agentlog.attach(bufnr, { source = "codex" })
    agentlog.attach(bufnr, { source = "cursor" })
    agentlog.refresh(bufnr)
    agentlog.detach(bufnr)
    agentlog.is_attached(bufnr)
    agentlog.get_document(bufnr)
    agentlog.goto_next("action", bufnr, count)
    agentlog.goto_previous("diff", bufnr, count)
    agentlog.open_file(bufnr)
<
`bufnr` may be 0 or omitted where applicable to target the current buffer.
Without `source`, attach detects the best matching enabled adapter. Explicit
selection remains available for ambiguous or truncated input. `get_document()`
returns the structured region document generated by the active adapter.
Navigation functions return the zero-based destination row, or nil plus a
message when no destination exists. `open_file()` returns the resolved path.

==============================================================================
CURRENT SCOPE                                                    *agentlog-scope*

The current implementation supports Codex, Claude Code, and Cursor Agent
adapters and the Zellij scrollback workflow. Recognition is deliberately
conservative: ambiguous text stays neutral instead of receiving a potentially
incorrect semantic type. Cursor's unmarked later conversational turns remain
neutral unless surrounding tool activity proves their boundary.

Navigation currently covers actions, changed diff blocks, explicit hunks,
assistant responses, recognized files, errors, and warnings. Folding, copy
commands, and focus modes are not yet implemented. Configuration fields reserved
for those features should not be considered active until their corresponding
behavior is documented here.
